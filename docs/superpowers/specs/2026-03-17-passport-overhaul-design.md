# Passport Overhaul — Design Spec

## Overview

Transform the passport from a basic stamp grid into a rich, immersive travel document. Five phases: visa pages, texture/feel, achievements tab, AI mission debriefs from photos, and travel intel summary.

## Current State

- SVG stamps generated by Claude Haiku (guilloche, arced text, landmarks)
- PassportCover with compass rose, gent info, stats
- StampGrid with 3-column layout, tab filtering (All/Missions/Achievements/Diplomatic)
- StampDetail modal showing stamp image + metadata
- Achievements tab is an empty div
- No visa pages, no texture, no AI narratives

---

## Phase 1: Visa Pages

### Trigger
Tap a mission stamp in StampGrid → instead of the current StampDetail modal, open a full visa page.

### Layout
Full-screen page (not a modal) at route `/passport/visa/{stampId}`:
- **Background**: aged paper CSS texture (warm ivory gradient with subtle noise)
- **Stamp**: SVG stamp image positioned top-right, rotated -8deg, ~120px
- **Header**: "MISSION DOSSIER" in monospace tracking-widest, gold-muted
- **Title**: entry.title in Playfair Display, large
- **Date**: formatted date range or single date, monospace
- **Location**: city, country with flag emoji, monospace
- **Participants**: small avatar row of who was there
- **Lore**: the 2-sentence lore in italic, as a quick summary
- **Mission Debrief**: the full AI-generated narrative (Phase 4), if available. Otherwise a "Generate Debrief" button.
- **Photo strip**: horizontal scroll of up to 6 entry photos
- **Landmarks**: if `metadata.landmarks` exists, show as tagged pills
- **Footer**: "View Full Entry" button linking to `/chronicle/{entryId}`

### Data
- Fetch entry by `stamp.entry_id` via `fetchEntry(id)`
- Fetch entry photos via `fetchEntryPhotos(entryId)`
- Participants come from `entry.participants`

### Navigation
- Back button returns to passport stamps view
- For non-mission stamps (achievements, diplomatic), keep the existing StampDetail modal

### File
- New page: `src/pages/VisaPage.tsx`
- Route: `/passport/visa/:stampId` in App.tsx

## Phase 2: Passport Texture & Feel

### Stamp Grid Background
- Warm aged paper background on the stamps view: CSS gradient `linear-gradient(135deg, #1a1610 0%, #0f0d0a 50%, #1a1610 100%)` with a subtle noise overlay via a tiny inline SVG `<filter>` element
- Faint page border: `border border-gold/8 rounded-xl` on the grid container
- Between stamp rows: decorative gold hairline dividers at low opacity

### Passport Cover Enhancements
- Add subtle leather texture via CSS: `repeating-linear-gradient` with very slight color variation
- Add a faint embossed border effect: `box-shadow: inset 0 0 60px rgba(201,168,76,0.05)`

### Stamp Detail Visual
- When stamp detail modal opens (for non-mission stamps), add the aged paper background
- Stamp image rendered larger (160px) with a subtle drop shadow

- Modify: `src/components/passport/PassportCover.tsx` — add CSS texture
- Modify: `src/components/passport/StampGrid.tsx` — add paper background

## Phase 4: AI Mission Debrief from Photos

### Concept
A rich, classified-style narrative generated from mission photos by Claude Sonnet with vision. Much deeper than the 2-sentence lore — this is the full story of the mission.

### Edge Function: `generate-mission-debrief`
- **Model**: `claude-sonnet-4-6` with vision
- **Input**: entry data (title, date, city, country, participants) + up to 20 photo URLs
- **Prompt**: Analyze all photos as chapters of a mission. Identify:
  - Locations, landmarks, venues visited
  - Food and drinks consumed
  - Activities and events
  - Mood, energy, weather, time of day
  - Who is present (using gent identities from `_shared/gent-identities.ts`)
- **Output** (structured):
  - `debrief`: 2-3 paragraph classified narrative in diplomatic language ("MISSION DEBRIEF: BUDAPEST PROTOCOL. Operatives deployed to the Hungarian capital...")
  - `landmarks`: string[] of identified landmarks/venues
  - `highlights`: string[] of 3-5 key moments
  - `risk_assessment`: tongue-in-cheek 1-sentence assessment ("Culinary exposure: severe. Budget impact: critical.")
- **Storage**: saved to `entry.metadata.mission_debrief`, `entry.metadata.landmarks`, `entry.metadata.debrief_highlights`, `entry.metadata.risk_assessment`

### UI Integration
- "Generate Debrief" button on visa page (Phase 1)
- Shows spinner during generation
- Once generated, displays on visa page:
  - Debrief text in body font, justified
  - Landmarks as gold-bordered pills
  - Highlights as a numbered list
  - Risk assessment in a bordered box at the bottom
- Can regenerate (replaces existing debrief)

### Gent Identity
- Import `GENT_VISUAL_ID` and `GENT_APPEARANCES` from `_shared/gent-identities.ts`
- Pass to prompt so Claude identifies each gent by name

### Files
- New edge function: `supabase/functions/generate-mission-debrief/index.ts`
- New client function: `src/ai/debrief.ts`
- Modify: `src/pages/VisaPage.tsx` — debrief section + generate button

## Phase 5: Travel Intel Summary

### Location
Passport cover page, below the stats box.

### Content
AI-generated 2-3 sentence intelligence briefing about the gent's travel patterns:
- Most visited city + frequency
- Total countries explored
- Travel style observations (e.g., "Subject shows preference for Balkan operations")
- Derived from entry stats (type counts, city frequency, date patterns)

### Generation
- Claude Haiku (fast, cheap)
- Input: gent stats from `fetchGentStats()` + mission city list
- Cached in `localStorage` key `codex_travel_intel_{gentId}`
- Regenerated if older than 7 days or on manual tap
- Fallback: shows stats without AI text if generation fails

### Files
- New client function: `src/ai/travelIntel.ts`
- Modify: `src/components/passport/PassportCover.tsx` — add intel section

## Phase Order & Dependencies

```
Phase 1 (Visa Pages) — standalone, no deps
Phase 2 (Texture) — standalone, no deps
Phase 3 (Achievements) — standalone, no deps
Phase 4 (AI Debriefs) — depends on Phase 1 (visa page is the display surface)
Phase 5 (Travel Intel) — standalone, no deps
```

Phases 1, 2, 3, 5 can be built in parallel. Phase 4 requires Phase 1.
