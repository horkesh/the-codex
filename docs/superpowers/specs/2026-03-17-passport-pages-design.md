# Passport Pages — Design Spec

## Overview

Rebuild the digital passport to match the physical "Pasoš" design. Three page types rendered as React components, viewable in the passport section and exportable from Studio.

## Reference

Physical passport: `docs/Pasoš - Haris (8.8 x 12.5 cm).pdf` (48 pages)
Reference images extracted to `docs/passport-ref/`

---

## Asset: Passport Cover

The physical cover image (`public/passport-cover.png`) is used as the passport cover background. The existing `PassportCover.tsx` component replaces its CSS compass-rose design with this image, overlaying personalized gent info (name, alias, avatar) on top.

**Personalization overlay:**
- Gent's display name (bottom area, where "PASOŠ" text is)
- Gent's alias below
- Gent's avatar (small, positioned near the emblem)

The cover image itself is static — no need to regenerate it. The personalization is CSS-overlay text on the image.

## Page Type 1: Visa Page

Matches the physical visa pages (pp 6-17 in the PDF). Each mission entry gets one.

### Layout (portrait, 4:5 ratio for export — 1080x1350px)

**Header bar** (top of page, outside the visa frame):
- "VIZE-ВИЗЕ-VISAS" in small caps, navy blue, centered

**Visa frame** (main content, cream/ivory background with guilloche border):
- **Guilloche border**: CSS/SVG ornate border around the entire visa. Green-tinted repeating wave pattern. Can be a static SVG border asset or CSS-generated.
- **Country emblem**: top-left corner, small (40-50px). Use a flag emoji or a simple SVG crest. For known countries (Croatia, Serbia, Bosnia, Hungary, Italy, Slovenia), use specific styling.
- **"VIZA" / "ENTRY VISA" header**: Large serif text, navy blue. Use country-appropriate language:
  - Croatia → "VIZA"
  - Serbia → "ВИЗА" (Cyrillic)
  - Bosnia → "VIZA"
  - Hungary → "ENTRY VISA"
  - Italy → "ENTRY VISA"
  - Default → "ENTRY VISA"
- **Cover photo**: tilted ~5-8deg, with a white border (polaroid style) and subtle tape effect (small translucent strip across top corner). Uses the entry's `cover_image_url`.
- **Country map silhouette**: faint grey outline of the country behind the data fields. Can be a simple SVG or omitted if too complex.
- **Data fields** (left-aligned labels, right-aligned values):
  - `DESTINATION:` → city name
  - `DATE OF TRIP:` → formatted date (MONTH YEAR)
  - `NUMBER OF GENTS:` → participant count (e.g., "3" or "3+1")
- **Mission stamp**: bottom area, rotated slightly. The SVG stamp already generated by Claude. If no stamp image, generate a text-based stamp with mission title + date.
- **Watermark**: faint Europe map outline behind everything (very low opacity)

### Data
- Entry: title, date, city, country, country_code, cover_image_url, participants
- Stamp: from `passport_stamps` linked via entry_id
- Photos: cover image from entry

### Component
`src/export/templates/VisaStampPage.tsx` — a Studio-exportable template (1080x1350px, same pattern as other templates with `ref` forwarding).

## Page Type 2: Notes/Debrief Page

Matches the physical "BILJEŠKE-ЗАБЕЛЕЖКЕ-OBSERVATIONS" page (p4 in the PDF). Used for the mission debrief text.

### Layout (portrait, 4:5 ratio — 1080x1350px)

**Header**: "BILJEŠKE-ЗАБЕЛЕЖКЕ-OBSERVATIONS" in small caps, navy blue, centered at top

**Background**: cream/ivory with green guilloche border (same as visa page) and faint Europe map watermark

**Content**:
- **Mission title** in display font, navy, centered
- **City, Country** below title
- **Date** in monospace
- **Debrief text**: the `mission_debrief` from metadata, in body font, justified, navy text on cream background
- **Landmarks**: listed below debrief as inline text
- **Risk Assessment**: in a bordered box at bottom
- **Page footer**: "THE GENTS CHRONICLES" small text, centered

### Data
- Entry metadata: `mission_debrief`, `landmarks`, `debrief_highlights`, `risk_assessment`
- Entry: title, date, city, country

### Component
`src/export/templates/DebriefPage.tsx` — Studio-exportable template.

## Page Type 3: ID Page

Matches the physical ID page (p5 in the PDF). One per gent — their personal passport identity page.

### Layout (portrait, 4:5 ratio — 1080x1350px)

**Background**: cream/ivory with green guilloche border

**Content**:
- **Portrait**: gent's avatar in a circular frame with "LORE KEEPER" / "BEARD & BASS" / "KEYS & COCKTAILS" text banner around it
- **Data fields**:
  - `NAME:` → gent's real name (from `display_name`)
  - `GENT'S STATION:` → alias (Lore Keeper, Beard & Bass, Keys & Cocktails)
  - `SIGNATURE DRINK:` → stored in gent profile or hardcoded
  - `INITIATION TRIP:` → first mission entry date + city
  - `ISSUE DATE:` → app join date or current date
- **QR code**: bottom-right, linking to the gent's profile page
- **BrandMark**: "THE.GENTS.CHRONICLES" bottom center

### Component
`src/export/templates/PassportIdPage.tsx` — Studio-exportable template.

## Passport Inner Page Flow

When the user opens the passport (taps cover → stamp view), the experience should feel like flipping through pages:

1. **Cover** (existing PassportCover with real cover image)
2. **ID Page** (gent's identity — viewable, exportable)
3. **Visa Pages** (one per mission — tap stamp → view visa page)
4. **Notes Pages** (debrief — accessible from visa page or separately)
5. **Achievements** (existing tab, already implemented)

### Navigation
- Stamp grid remains the primary view inside the passport
- Tapping a mission stamp → opens the Visa Page (already implemented as VisaPage.tsx, needs visual redesign)
- "View Debrief" button on visa page → shows notes page
- Both visa and notes pages exportable from Studio

## Visual Assets Needed

| Asset | Source | Format |
|---|---|---|
| Passport cover | Extracted from PDF p1 | `public/passport-cover.png` (already done) |
| Guilloche border | CSS/SVG generated | Inline SVG in components |
| Europe map watermark | SVG trace of the faint map | Inline SVG, very low opacity |
| Country emblems | Flag emoji or simple SVG | Inline per-country |

### Guilloche Border Approach
The physical passport uses an intricate green guilloche pattern. For the digital version:
- Use a CSS border pattern: repeating sine-wave SVG `<pattern>` element
- Color: `rgba(100, 160, 120, 0.15)` — faint green
- Applied as an SVG `<rect>` frame around the visa content area
- Simpler than the physical version but evokes the same feel

### Europe Map Watermark
- Simple SVG outline of Europe
- Color: `rgba(100, 160, 120, 0.08)` — barely visible
- Centered in the content area behind text

- No multi-page PDF export (one page at a time via Studio)
- No country-specific map silhouettes (too many to maintain — use the Europe watermark for all)
